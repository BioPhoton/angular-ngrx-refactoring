Types of state:

Server State

The server state is stored on the server and provided over a REST-API endpoint. (www.angular.at/api)

Persistent State

The persistent state is a subset of the server state stored on the client. It can be stored in a cache object, in the local stored or anywhere else.
It is a good practice to reflect this state in the url. i.e. bookmark

Client State

The client state is only stored in memory in a non persistent way i. e. the values in form inputs
It is a good practice to reflect this state in the url. i.e. bookmark

Router/URL state

The router/URL state is present in the browser search bar and as a structural representation in our code.

Transient Client State

The transient client state describes some sort of client state that is visible but not reflected in the url.
A example would be the scroll position of your window or the time bar in a video player.

Local UI State

The local UI state on the compoent level. It describes some component internal UI state.. i.e. a progress spinner, collapsed state of accordion.
